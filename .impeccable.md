## Design Context

### Users
效率型和收藏型用户，日常使用 Chrome 浏览器工作/学习/生活，书签量从几十到几百不等。他们在每次打开新标签页时使用 MarkPage — 这是一个极高频的触点，要求即时响应。使用场景涵盖白天办公和夜间休闲浏览。

### Brand Personality
**温暖、灵动、友好** — 像一个懂你习惯的聪明助手，安静但总在你需要时出现。不冷漠但也不聒噪。效率驱动但有温度。

- 声音：平和、自信、略带幽默
- 情感目标：让用户感到"掌控感"和"被理解"
- 反面参考：不要像 Momentum 那样花哨臃肿，不要像 Chrome 原生那样冷漠无趣

### Aesthetic Direction
- **视觉基调**：Raycast/Spotlight 式搜索中心体验，但注入温暖和有机感。搜索框是灵魂，一切围绕它展开
- **参考**：Raycast 的高效命令面板美学 + Things 3 的温暖克制
- **反参考**：不要 Momentum 的壁纸+励志名言套路，不要过度毛玻璃/发光效果
- **主题**：跟随系统 + 双模式（light/dark），两种模式都需要精心设计
- **配色**：基于品牌调性自由发挥，方向是温暖但不幼稚

### Design Principles
1. **搜索即入口** — 搜索框是视觉焦点和交互起点，一切围绕它组织
2. **温暖的效率** — 功能至上但有温度，每个元素都服务于快速找到目标
3. **渐进式展现** — 首屏极简，复杂功能通过交互层层展开
4. **尊重注意力** — 新标签页打开频率极高，不抢夺注意力，做安静的工具
5. **可感知的品质** — 微妙的动效、精准的间距、一致的视觉语言传递专业感
